Architecture Approaches

How should Tauri, the Preact dashboard, and cloudmock's Go binary fit together?

A

Tauri Shell + Embedded Dashboard (Recommended)

Tauri's Rust backend spawns cloudmock's Go binary as a child process and manages its lifecycle. The existing Preact dashboard is served by cloudmock on port 4500 as it does today, and Tauri's webview loads it directly. Tauri adds native window chrome, system tray, and IPC commands for start/stop/config.

Architecture
Tauri App
Rust backend (process mgmt, IPC)
Webview → localhost:4500
↕
cloudmock gateway (Go binary)
:4566 AWS API · :4500 Dashboard · :4599 Admin

Pros

  • Zero changes to existing dashboard code
  • Existing dashboard features work immediately
  • Clean process boundary — Go binary is independent
  • Can still use dashboard in browser if preferred

Cons

  • Two processes to manage (Tauri + Go)
  • Port conflicts possible if cloudmock already running
  • Adding new native panels means new Tauri webview windows or tabs
B

Tauri Shell + Rebuilt Frontend

Tauri spawns cloudmock's Go binary for the backend, but replaces the Preact dashboard with a new frontend built specifically for the desktop app. The new UI talks to cloudmock's admin API (:4599) and adds device debugging panels natively. Uses a modern framework like SolidJS or Svelte for performance.

Architecture
Tauri App
Rust backend + New SPA frontend
Talks to admin API on :4599
↕
cloudmock gateway (Go, headless)
:4566 AWS API · :4599 Admin API only

Pros

  • Purpose-built UI for desktop (not a wrapped web page)
  • Can integrate device panels seamlessly from day 1
  • Full control over UX without web-dashboard constraints

Cons

  • Massive upfront effort — rewrite 28 dashboard pages
  • Two frontends to maintain (web dashboard + desktop)
  • Delays MVP significantly
C

Tauri Hybrid — Embed Existing + Extend

Start with Approach A (embed existing dashboard), but build new panels (device debugging, log aggregation) as separate Tauri-native views using the same Preact/Vite stack. The Tauri app becomes a tabbed container: "Cloud" tab loads cloudmock dashboard, "Devices" tab loads new debugging UI, "Logs" tab shows unified log stream.

Architecture
Tauri App
Rust backend (process mgmt, device bridges)
Tab: Cloud → cloudmock dashboard (:4500)
Tab: Devices → new Preact views (bundled)
Tab: Logs → unified log stream (bundled)
↕
cloudmock
Go binary
Device bridges
ADB, xcrun, CDP

Pros

  • Fast MVP — existing dashboard works day 1
  • New panels built purpose-fit for desktop
  • Incremental migration path — can replace cloudmock dashboard pages one by one
  • Same tech stack (Preact/Vite) across all views

Cons

  • Some UX inconsistency between embedded and new views initially
  • Slightly more complex routing (multi-webview or iframe-based tabs)
