# Dashboard 2.5: New Panels for Phases 3-6 — Design Specification

**Date:** 2026-03-23
**Status:** Approved
**Depends on:** Phases 3-6 backend APIs

---

## Overview

Add dashboard coverage for all backend APIs built in Phases 3-6: incidents, regressions, trace comparison, cost intelligence, profiling, audit, webhooks, and user management. Three new sidebar pages, four integrated additions to existing pages.

### Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Organization | Hybrid | Incidents/regressions as top-level pages; cost/profiling integrated; admin in Settings |
| Incidents layout | Summary cards + timeline + table | Instant situational awareness + detail |
| Trace comparison | Button in trace detail + URL route | Natural entry point, no sidebar clutter |

---

## 1. New Sidebar Entries

Add between existing items:
- **Incidents** (`#/incidents`) — after Traces
- **Regressions** (`#/regressions`) — after Incidents
- **Settings** (`#/settings`) — at bottom

---

## 2. Incidents Page

**Top:** 4 summary cards — Active (red if >0), Critical, Warning, Resolved today

**Middle:** 24h timeline sparkline

**Bottom:** Filterable table — Severity badge, Title, Services, Alert Count, First Seen, Status. Filters: status, severity, service. Click row → expand with root cause, regressions, timeline, owner. Actions: Acknowledge, Resolve, Export Report (HTML).

API: `GET /api/incidents`, `POST /api/incidents/{id}/acknowledge`, `POST /api/incidents/{id}/resolve`, `GET /api/incidents/{id}/report?format=html`

---

## 3. Regressions Page

**Top:** Summary cards — Active count, by severity, by algorithm type

**Bottom:** Filterable table — Severity badge, Algorithm pill, Service, Change %, Confidence bar, Deploy, Detected. Filters: service, severity, algorithm, status. Click row → expand with before/after values, windows, dismiss button.

API: `GET /api/regressions`, `POST /api/regressions/{id}/dismiss`

---

## 4. Settings Page

Tabbed layout:
- **Webhooks** — CRUD table, test button per webhook. API: `GET/POST/DELETE /api/webhooks`, `POST /api/webhooks/{id}/test`
- **Users** (auth-enabled only) — user list, role dropdowns. API: `GET /api/users`, `PUT /api/users/{id}`
- **Audit** — filterable log table. API: `GET /api/audit`

---

## 5. Trace Comparison

Button in trace detail view → dropdown: "Compare with trace" (enter ID) or "Compare with baseline".

**Route:** `#/traces/compare?a={id}&b={id}` or `?a={id}&baseline=true`

**Layout:** Summary bar (latency A vs B, counts) + side-by-side matched spans with latency deltas (green=faster, red=slower) + added/removed spans highlighted.

API: `GET /api/traces/compare?a={id}&b={id}` or `?a={id}&baseline=true`

---

## 6. Cost Tab in Metrics

New tab alongside existing metrics:
- Top routes by cost (table)
- Top tenants by cost (table)
- Cost trend line chart (24h or 7d)

API: `GET /api/cost/routes`, `GET /api/cost/tenants`, `GET /api/cost/trend?window=7d&bucket=1h`

---

## 7. Profile Tab in ServiceInspector

New tab in the right panel:
- Capture CPU / Heap / Goroutine buttons
- Recent profiles list
- Click → simple flame graph (CSS stacked bars from folded stacks format)

API: `POST /api/profile/{service}?type=cpu&duration=5s`, `GET /api/profiles?service=X`, `GET /api/profiles/{id}?format=flamegraph`
