View Inventory — What Each Icon Does

⚡ Activity
Unified stream from all sources. Logs, network requests, errors, cloudmock API calls — interleaved chronologically. Filter by source, level, type. Click any event → inspector panel.
🗺️ Topology
Service graph from existing console. Nodes = your apps + cloudmock services. Edges = traffic between them (auto-discovered from request correlation). Health-colored, traffic-sized. Shows cross-platform data flow.
☁️ Services
Resource browser for cloud services — S3 buckets, DDB tables, Lambda functions, SQS queues, Cognito pools, etc. Same as existing dashboard's resource pages but rebuilt for desktop. Also shows K8s + ArgoCD resources.
🔍 Traces
Distributed trace waterfall from existing console. Now includes spans from client apps too — see the full journey from button tap → HTTP request → Lambda → DynamoDB → response.
📊 Metrics
Request volume, latency percentiles, error rates, SLO burn rates — from existing console. Now includes client-side metrics (app launch time, frame rate, crash rate) per source.
🎯 SLOs
SLO dashboard from existing console — burn rate, error budgets, compliance. Extended to track client-side SLOs (e.g., "95% of Flutter API calls complete under 500ms").
🚨 Incidents
Incident management from existing console — alert grouping, timeline, regression detection. Now surfaces client-side crashes and errors alongside server-side incidents.
🔥 Profiler
Flame graphs, CPU/heap profiling from existing console. Extended: Flutter DevTools performance overlay, React Native Hermes profiling, Chrome DevTools CPU profiler — all in one flame graph view.
🧪 Chaos
Chaos engineering from existing console — inject latency, errors, throttling into cloudmock services. Test how your Flutter/RN/Web apps handle degraded backends.
🤖 AI Debug
AI explain from existing console — select any request, trace, or error and get a narrative explanation. Now with full context: "Your Flutter app's S3 upload failed because the IAM policy doesn't allow PutObject on the uploads/ prefix."

Key Principle

Every view shows data from all sources. Topology shows your Flutter app talking to S3. Traces show the journey from button tap to DynamoDB. Metrics include client-side latency. There are no "cloud-only" or "device-only" views — the whole point is correlation across the stack.
