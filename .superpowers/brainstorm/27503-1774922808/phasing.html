Product Phasing

From MVP to full platform — what ships when

Phase 1 Desktop App MVP Build first
Tauri shell + new Preact frontend + local cloudmock management. The "it works" moment.
Tauri Shell
  • Window with icon rail + resizable panels
  • System tray with start/stop
  • Native menus, keyboard shortcuts
  • Connection picker (local only)
Rust Backend
  • Process manager (spawn/stop Go binary)
  • Health monitoring + auto-restart
  • Config file read/write
  • Admin API proxy (IPC → HTTP)
Cloud Services Panel
  • Service list with health status
  • Per-service resource browser (S3 buckets, DDB tables, etc.)
  • Service detail view with stats
Request Log Panel
  • Live SSE stream of API calls
  • Filter by service, status, action
  • Request detail inspector
  • Timestamp, latency, response body
Phase 2 Observability + Topology
Port the existing console's best features into desktop-native panels.
Topology Canvas
  • Service graph with health-colored nodes
  • Traffic-sized edges
  • Click node → detail panel
Traces + AI Explain
  • Distributed trace waterfall
  • Span detail with timing
  • AI-powered request explanation
SLOs + Alerts
  • SLO burn rate dashboard
  • Native notifications on violations
  • Incident timeline
Phase 3 Device Debugging
The Flipper replacement — cross-platform device inspection for Flutter, React Native, and Web.
Device Manager
  • Auto-detect iOS sims + Android emulators
  • ADB + xcrun/idb bridges in Rust
  • Device list with status
  • Launch/kill simulators
Unified Log Viewer
  • iOS os_log + Android logcat + console.log
  • Filter by device, level, tag
  • Interleaved timeline
  • Flutter/Dart log parsing
Network Inspector
  • HTTP requests from all platforms
  • Correlate with cloudmock request log
  • Request/response body viewer
  • CDP + Flutter DevTools protocol
Phase 4 cloudmock.io + Hosted Platform
The SaaS layer — hosted instances, team management, billing.
cloudmock.io Web
  • Marketing landing page
  • Account + team management
  • Instance provisioning
  • API key management
  • Usage dashboards + billing
Hosted Backend
  • Container orchestration (ECS/K8s)
  • Per-tenant isolated instances
  • Persistent state (PostgreSQL/DuckDB)
  • Auth layer + API key validation
Desktop Integration
  • Login with cloudmock.io account
  • Connection picker: hosted instances
  • Team-shared saved views
  • Sync config across machines
Phase 5 Developer Portal + Ecosystem
Service catalog, API docs browser, plugin marketplace, community plugins for GCP/Azure emulation, third-party integrations (Datadog, PagerDuty, Slack).
