How should the view toggle work?

Switching between runtime flow and dependency tree in the topology view

A

Tab bar inside topology view

Two tabs at the top of the existing topology view: "Runtime Flow" and "Dependencies". Same view, same nav item, two modes. The graph re-layouts when you switch but keeps the same canvas, minimap, and inspector sidebar.

B

Separate nav item in sidebar

A new "Dependencies" entry in the icon rail under the "Observe" group, next to Topology. Completely separate view with its own state, layout, and inspector. Can have both open in memory and switch instantly.

Dropdown in topology toolbar

A layout picker dropdown in the topology view's toolbar: Layered → Tree → Force. Same data, different ELK algorithm. "Tree" uses the IaC hierarchy for parent-child ordering. Lightweight, no new view needed.
