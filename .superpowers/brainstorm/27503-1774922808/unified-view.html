Key Concept: Sources, Not Silos

Everything is a "source" — your Flutter app, React Native app, web app, Node server, and cloudmock are all sources that feed into one unified activity stream. No separation by platform.
Automatic correlation — when your Flutter app makes an HTTP request that hits cloudmock S3, the devtools automatically links the client-side request to the server-side action. You see both sides of the call in the inspector.
Filter, don't tab-switch — want to see only Flutter errors? Filter the stream. Want only network requests? Filter. Want only cloudmock Lambda invocations? Filter. The data is always in context, you just narrow your view.
Icon rail is for views, not platforms — 🏠 Workspace (the unified stream), 🗺️ Topology (service graph), 🔥 Profiler (flame graphs), ⚙️ Settings. Each view shows data from ALL sources.
