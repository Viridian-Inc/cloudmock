Implementation Phases

Phase A: Live Service Map First

  • Health state computation (green/yellow/red from /api/metrics)
  • Node coloring + user-facing impact detection (path BFS)
  • Edge metrics (req/s, p99) from topology callCount/avgLatencyMs
  • Incident badges + deploy markers on nodes
  • Polling hooks (metrics every 10s, topology/deploys/incidents every 30s)

Phase B: Inspector Tabs

  • Metrics tab with sparkline charts (req/s, p99, error rate)
  • Endpoints tab from service-manifest.json + OpenAPI scan
  • Deploys tab from /api/deploys
  • Incidents tab with acknowledge/resolve
  • Connections tab with blast radius count

Phase C: Timeline + Time Travel

  • Timeline bar (deploys, incidents, alerts)
  • Time range selector (Live, 15m, 1h, 6h, 24h, 3d, 7d)
  • Historical scrubbing — map shows past health states
  • SQLite persistence layer in Rust (7-day retention)

Phase D: Service Browser + Routing

  • Domain-grouped service list (explicit config)
  • Local/cloud routing toggle per service
  • OpenAPI spec file scanning
  • Domain config file (service-domains.json)
