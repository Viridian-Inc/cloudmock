## Methodology

**Harness:** A Go binary using AWS SDK v2 exercises all providers with identical code paths. No provider-specific workarounds.

**Iterations:** 50 warm iterations per operation after a cold-start measurement. Concurrency tested at 5 goroutines.

**Metrics:**
- **P50/P95/P99** latency measured over warm iterations
- **Startup time** — wall clock from `docker run` to first successful health check
- **Memory** — peak RSS sampled via `/proc/<pid>/status` (Linux) or `ps` (macOS) after all operations complete
- **Correctness** — response schema validation and AWS error code verification per operation

**Environment:** Docker containers on Apple Silicon (darwin/arm64). CloudMock and LocalStack run as containers. Moto runs via `moto_server` in a container. All containers use the same CPU and memory limits.

**Date:** April 2, 2026 — Go 1.26.1, darwin/arm64.

**Reproduce it yourself:**

```bash
# Full benchmark (requires Docker)
go run ./benchmarks/cmd/bench --target=all --mode=docker

# Quick run, specific services
go run ./benchmarks/cmd/bench --target=cloudmock --services=s3,dynamodb,sqs --quick
```

Results are written to `benchmarks/results/` as JSON.
