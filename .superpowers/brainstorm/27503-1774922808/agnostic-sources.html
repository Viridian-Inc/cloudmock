What About Flutter (Dart)?

Dart is a 4th runtime — add it if there's demand. Dart's HttpClient, dart:developer, and zone error handling are distinct from Node/Swift/Kotlin. If you want Flutter support, a Dart SDK is a clean addition to the same pattern. But the architecture doesn't depend on it — it's just another source.

No-SDK Alternatives Still Work

Proxy mode
Route traffic through devtools as a proxy. Works with any language, any framework, zero code changes. Network only (no logs/crashes).
cloudmock correlation
Even without an SDK, cloudmock's own request log shows all API calls. You just lose the client-side perspective (source file, client timing, logs).
