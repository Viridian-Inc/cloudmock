How It Fits Together

Shared API surface — Both local and hosted cloudmock expose the same admin API. The desktop app doesn't care whether it's talking to localhost:4599 or api.cloudmock.io — it just needs an endpoint URL.
Desktop = full experience — Cloud services + device debugging + network monitoring + log aggregation. Works with local or hosted cloudmock.
Web = management console — cloudmock.io is the marketing site + web console for provisioning instances, managing API keys, viewing usage/billing. Not a debugging tool.
Auth layer — Hosted mode requires authentication. The desktop app stores an API key (from cloudmock.io console) to connect to hosted instances. Local mode stays unauthenticated.

Revised MVP Scope

Phase 1 — Desktop app with local cloudmock

  • Tauri shell, system tray, native menus
  • Start/stop/restart local cloudmock gateway
  • Service overview, live request log, config UI
  • Connection settings (endpoint URL — defaults to localhost)

Phase 2 — Hosted backend + cloudmock.io

  • cloudmock.io landing + web console
  • Hosted cloudmock instances (container orchestration)
  • API key auth, desktop app connects to hosted endpoint

Phase 3 — Device debugging

  • ADB/iOS/CDP bridges in Rust
  • Unified log viewer, network inspector, view hierarchy
