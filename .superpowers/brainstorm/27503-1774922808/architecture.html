Key Design Decisions

Headless cloudmock — The Go binary runs without its own dashboard (:4500 disabled). The Tauri frontend talks directly to the admin API on :4599. This avoids port conflicts and gives us full UI control.
Same tech stack as existing dashboard — Preact + Vite + TypeScript. This means existing dashboard components can be migrated (not rewritten from scratch) into the new app as needed. The API client code is directly reusable.
Rust-native device bridges — ADB, xcrun/idb, and Chrome DevTools Protocol connections live in Rust. The frontend gets device data via Tauri IPC commands and events, not HTTP polling.
Event-driven architecture — Tauri v2 events push real-time data (log lines, device state changes, network requests) to the frontend. No polling loops.

MVP Scope (Phase 1)

  • Tauri app shell — window, system tray, native menus
  • Process manager — start/stop/restart cloudmock gateway from the app
  • Service overview — list of running AWS services with health status (from :4599/api/services)
  • Request log — live stream of AWS API calls (from :4599/api/requests + SSE)
  • Basic config UI — region, profile, IAM mode selection

This gets a working desktop app shipping fast. Device debugging comes in Phase 2+.
